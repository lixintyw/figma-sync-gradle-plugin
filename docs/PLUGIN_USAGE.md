# Figma Sync Gradle Plugin — Usage Guide

## Quick Start

```kotlin
// build.gradle.kts
plugins {
    id("com.figma.sync")
}

figmaSync {
    fileKey = "G4GyegR4f1uHsW5ZdBuOaY" // Figma file key from URL
    // Token is read from local.properties: figma.token=xxx

    icons {
        enabled = true
        startNode = "14832:59978"   // Figma node ID of the root Section/Frame
        scale = 2                    // 1, 2, 3, or 4
        declarations = listOf("icon_car_door", "icon_window")
        extractTokens = true         // Also generate token bindings
    }

    tokens {
        enabled = true
        chainDownload = true         // Auto-resolve token alias chains
    }
}
```

## Tasks

| Task | Description |
|------|-------------|
| `syncFigmaIcons` | Download SVG icons → VectorDrawable XML |
| `syncFigmaTokens` | Fetch design tokens → `figma_colors.xml` + `token_bindings.json` |
| `syncFigmaColors` | Alias for `syncFigmaTokens` |
| `syncFigmaAll` | Run all sync tasks |
| `figmaFreezeVersion -PfreezeName="HMI v2.1"` | Snapshot current cache as named version |
| `figmaRestoreVersion -PfreezeName="HMI v2.0"` | Restore a frozen version's cache |
| `figmaListVersions` | List all frozen versions |
| `figmaDiffVersions -PdiffFrom="v2.0" -PdiffTo="v2.1"` | Compare two versions |

## DSL Reference

### Top-level

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `token` | String | from `local.properties` | Figma personal access token |
| `fileKey` | String | (required) | Figma file key |
| `pinnedVersion` | String | `""` | Pin to a specific Figma version ID |

### `icons { }` block

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `enabled` | Boolean | `false` | Enable icon sync |
| `startNode` | String | `""` | Starting node ID in Figma tree |
| `scale` | Int | `2` | Render scale (1-4) |
| `declarations` | List\<String\> | `[]` | Icon names to download (empty = all) |
| `extractTokens` | Boolean | `false` | Generate `token_bindings.json` |

### `tokens { }` block

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `enabled` | Boolean | `false` | Enable token sync |
| `output` | String | `"figma_colors.xml"` | Output file name |
| `chainDownload` | Boolean | `true` | Resolve alias chains |
| `collections` | List\<String\> | `[]` | Filter by variable collections (empty = all) |
| `tokenSource` | String | `"rest_api"` | `"rest_api"` or `"plugin"` |
| `pluginExportFile` | String | `"src/main/assets/figma_plugin_export.json"` | Plugin JSON path |

### `modules { }` block

```kotlin
modules {
    module("vehicle_control") {
        enabled = true
        startNode = "14832:59978"
        scale = 2
        declarations = listOf("icon_car_door", "icon_window")
    }
    module("lighting") {
        enabled = true
        startNode = "14832:60000"
    }
}
```

Each module creates an independent task: `syncFigmaIcons_vehicleControl`, `syncFigmaIcons_lighting`.

## Output Files

| File | Path | Generated By |
|------|------|-------------|
| VectorDrawable (LTR) | `res/drawable/icon_*.xml` | `syncFigmaIcons` |
| VectorDrawable (RTL) | `res/drawable-ldrtl/icon_*.xml` | `syncFigmaIcons` |
| Color resources | `res/values/figma_colors.xml` | `syncFigmaTokens` |
| Token bindings | `assets/token_bindings.json` | `syncFigmaTokens` |
| Asset metadata | `assets/figma_assets.json` | `syncFigmaIcons` |

## Token Chain Download

When `chainDownload = true`, declaring token A automatically downloads all tokens that A references (VARIABLE_ALIAS resolution). This ensures complete token closure without manually listing every alias.

## Version Management

```bash
# Freeze a named version
./gradlew figmaFreezeVersion -PfreezeName="HMI v2.1"

# List all versions
./gradlew figmaListVersions

# Restore a previous version
./gradlew figmaRestoreVersion -PfreezeName="HMI v2.0"

# Compare two versions
./gradlew figmaDiffVersions -PdiffFrom="HMI v2.0" -PdiffTo="HMI v2.1"
```

Versions are stored in `build/.figma-versions/`. Each version is a snapshot of the cache file.

## Plugin Mode (Figma Plugin Fallback)

When the REST API Variables endpoint is unavailable (free Figma plan):

1. Designer exports token bindings using the Figma plugin (see FIGMA_PLUGIN_GUIDE.md)
2. Save the JSON to `app/src/main/assets/figma_plugin_export.json`
3. Configure:
   ```kotlin
   tokens {
       enabled = true
       tokenSource = "plugin"
   }
   ```
4. Run `./gradlew syncFigmaTokens`
